## Önceki Konuların Tekrarı

### Class ve Object

```csharp
class Player
{
    public string Name;
}
```

Bu kod object oluşturmaz; yalnızca tip tanımlar.

```csharp
Player player = new Player();
```

`new Player()` heap'te object oluşturur. `player`, bu object'e ulaşmayı sağlayan referans değişkenidir.

### Constructor

Constructor nesneyi allocate etmez. Heap allocation ve default field initialization sonrasında object'i valid duruma hazırlar.

### Field ve Property

- Field: Verinin tutulduğu alan.
- Property: `get` ve `set` accessor'larıyla kontrollü erişim.

### Stack Frame

Stack frame, bir metodun geçici çalışma alanıdır. İçinde parametreler, local değişkenler, return address ve runtime bilgileri bulunabilir.

### Heap ve GC

Metot bitince stack frame kaldırılır. Heap object'i hemen silinmez. Ulaşılabilir referans kalmazsa GC onu uygun zamanda toplayabilir.

## Bugün İşlenen Ana Konu

# Method Parameter Passing

## 1. Value Type Parametre

```csharp
void Increase(int x)
{
    x++;
}

int number = 5;
Increase(number);
Console.WriteLine(number);
```

Çıktı:

```text
5
```

Çünkü metoda `number` değişkeni değil, içindeki `5` değerinin kopyası gider.

```text
Main Frame              Increase Frame
number = 5   --copy-->  x = 5
```

`x++`, yalnızca Increase frame içindeki kopyayı değiştirir.

---

## 2. Reference Type Parametre

```csharp
void ChangeName(Player p)
{
    p.Name = "Veli";
}

Player player = new Player();
player.Name = "Ali";
ChangeName(player);
```

Çıktı:

```text
Veli
```

Çünkü `player` değişkeninin tuttuğu referans değeri kopyalanır. `player` ve `p`, aynı heap object'ini gösterir.

```text
Main Frame                   ChangeName Frame
player ----reference copy--> p
       \                     /
        ----> Player #1 <----
```

---

## 3. Nesneyi Değiştirmek ve Referansı Değiştirmek

```csharp
p.Name = "Veli";
```

Heap'teki nesnenin içini değiştirir.

```csharp
p = new Player();
```

Yalnızca method frame içindeki `p` değişkeninin tuttuğu referansı değiştirir.

Normal parametre aktarımında çağıranın `player` değişkeni değişmez.

---

## 4. `ref` Neden Var?

Normal parametre aktarımında her zaman bir değer kopyası gider:

- Value type için gerçek değer kopyalanır.
- Reference type için referans değeri kopyalanır.

Bazen metodun çağıranın değişkeninin kendisi üzerinde çalışması gerekir.

```csharp
void Swap(ref int a, ref int b)
{
    int temp = a;
    a = b;
    b = temp;
}
```

`ref`, çağıranın değişkeninin storage location'ına erişim sağlar.

```text
Swap Frame             Main Frame
ref a ---------------> x
ref b ---------------> y
```

`ref` boxing değildir. Value type'ı heap'e taşımaz ve object'e dönüştürmez.

---

## 5. Reference Type ile `ref`

```csharp
void SetPlayer(ref Player p)
{
    p = new Player();
}
```

Buradaki amaç object'in içini değiştirmek değildir. Çağırandaki referans değişkeninin hangi object'i gösterdiğini değiştirmektir.

```text
SetPlayer Frame               Main Frame
ref p ----------------------> player ----> Player #1
```

`p = new Player()` çağırandaki `player` değişkeninin referansını değiştirir.

---

## Bugünün Final Örneği

```csharp
void Change(ref Player p)
{
    p.Name = "Veli";
    p = new Player();
    p.Name = "Mehmet";
}

Player player = new Player();
player.Name = "Ali";

Change(ref player);
Console.WriteLine(player.Name);
```

Çıktı:

```text
Mehmet
```

Adımlar:

1. İlk object'in adı `Veli` yapılır.
2. `ref` üzerinden çağıranın `player` değişkeni yeni object'e yönlendirilir.
3. Yeni object'in adı `Mehmet` yapılır.

---

## Zorlanılan Noktalar

- Stack frame kavramı tekrar yapılmayınca unutuldu.
- `ref` için ilk tahmin boxing ile heap'e taşıma yönündeydi.
- Reference type ile `ref` kullanımında object'in mi, referans tutucunun mu değiştiği ayrıştırıldı.

---

## Güçlü Noktalar

- Reference type parametrede referans değerinin kopyalandığı doğru açıklandı.
- `p = new Player()` işleminin local referansı ezdiği doğru bulundu.
- `ref Player` örneğinde çağıranın değişkeninin referansının değişeceği doğru çözüldü.
- Final örneğin çıktısı `Mehmet` olarak doğru bulundu.

## Tekrar Kartları

### Kart 1

**Soru:** Class tanımı object oluşturur mu?

**Cevap:** Hayır. Object, `new` çalıştığında oluşur.

### Kart 2

**Soru:** Stack frame nedir?

**Cevap:** Bir metodun parametreleri, local değişkenleri ve dönüş bilgisi için kullanılan geçici çalışma alanıdır.

### Kart 3

**Soru:** `void Foo(int x)` çağrısında ne kopyalanır?

**Cevap:** `int` değeri kopyalanır.

### Kart 4

**Soru:** `void Foo(Player p)` çağrısında ne kopyalanır?

**Cevap:** Player object'i değil, referans değeri kopyalanır.

### Kart 5

**Soru:** `p.Name = "Veli"` neyi değiştirir?

**Cevap:** Heap'teki object'in içini değiştirir.

### Kart 6

**Soru:** `p = new Player()` neyi değiştirir?

**Cevap:** `p` referans değişkeninin hangi object'i gösterdiğini değiştirir.

### Kart 7

**Soru:** `ref` boxing yapar mı?

**Cevap:** Hayır. Çağıranın değişkeninin storage location'ına erişim sağlar.

### Kart 8

**Soru:** Reference type zaten referansken neden `ref Player` gerekir?

**Cevap:** Object'in içini değil, çağıranın referans değişkeninin tuttuğu referansı değiştirmek için.
